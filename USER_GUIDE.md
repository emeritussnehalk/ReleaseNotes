# Release Notes Data Explorer User Guide

## Overview

The Release Notes Data Explorer is a web-based viewer for release note records that can load from the local backend's scheduled Jira sync, with `data/master.csv` as a fallback source. It is designed to help business and operations users quickly find, filter, review, and export release-note entries without editing the source file.

The application displays release records in a searchable, paginated table and provides:

- Multi-select filters for key business fields
- A global search across the full table
- Clickable card-level detail view
- Built-in pagination
- Excel export for the currently filtered results

## What You See on the Screen

When the page loads successfully, the interface has three main areas:

1. Header
   Contains the branding, view toggle, and the global search box.

2. Filter bar
   Contains dropdown filters plus action buttons for resetting filters and downloading Excel.

3. Data table
   Shows the filtered release notes in rows and columns.

The data-source status appears as a small badge at the bottom-right of the page, with the Jira sync icon at the start.

## Protected Jira Sync

The app refreshes Jira data automatically when the server starts and on the 1st of each month at 10:00 AM local server time.

Use the sync icon in the bottom-right data-source badge only when you need an immediate refresh. The app asks for the authorized sync username and password before calling Jira.

## Data Shown in the Table

The table includes these columns:

- `Sr. No.`
- `Card Details`
- `Title`
- `Description`
- `University Name`
- `Program Name`
- `Business Unit`
- `Business Docs`
- `Tutorial Doc`
- `Skill Set`
- `Product Manager`
- `Business Impact`
- `Tech Team`
- `Release Date`

`Issue Type` exists in the data model but is hidden from the visible table.

## How `Sr. No.` Works

`Sr. No.` is generated by the application. It is not read from the CSV file.

- The numbering is based on the row position currently shown in the table.
- It updates automatically when filters, search, sorting, or pagination change the visible order.
- Page 2 continues from page 1 instead of restarting at 1.

## Default Data Order

When the app first loads the CSV, records are arranged in this order:

1. `Release Date` descending (newest first)
2. `Business Unit` ascending
3. `Business Impact` by priority: `High`, then `Medium`, then `Low`
4. `Card Details` / issue key ascending

After the table is loaded, users can still sort by clicking sortable column headers.

## Filters

The filter bar provides five dropdown filters:

- `University Name`
- `Program Name`
- `Business Unit`
- `Tech Team`
- `Release Month`

### How the filters work

- Each filter supports multiple selections.
- Open a filter and check one or more values.
- The table updates immediately after selection.
- If more than two values are selected, the filter button shows a count such as `3 selected`.
- `Select All` selects every value in that filter.

### Important default behavior

On first load, the app automatically selects the **previous month** in the `Release Month` filter if that month exists in the data.

Example:

- If today is April 2026, the default selected month is `Mar-2026`, if available in the CSV.

This means first-time users may initially see only one month of data until they clear or change that filter.

### Notes on specific filters

- `University Name` and `Program Name` can match multiple comma-separated values from a single row.
- `Tech Team` is sourced from the project field in the CSV and may display normalized values such as `SFDC` or `Portal`.
- `Release Month` is derived from the release date and grouped in `Mon-YYYY` format.

## Global Search

The search box in the top-right corner performs a table-wide text search.

- It searches across the visible dataset handled by DataTables.
- Search works together with the dropdown filters.
- The search is live, so results update as you type.

Use global search when you want to find:

- A keyword in the title or description
- A specific issue key
- A person name
- A business unit or team name

## Sorting

Users can sort table columns by clicking a column header.

- Sorting is handled by DataTables.
- `Sr. No.` is not sortable.
- Hidden columns such as `Issue Type` are not part of the visible interaction.
- `Release Date` sorts using the parsed internal date value, not just the displayed text.
- `Business Impact` sorts by logical priority rather than plain alphabetical order.

If a user sorts a column manually, the visible row order changes and `Sr. No.` updates to match that new order.

## Pagination

The table uses pagination to keep the interface responsive and easier to scan.

- Default page size is `100` rows.
- The bottom pagination bar shows:
  - rows-per-page selector
  - current record summary
  - page navigation buttons
- Available page sizes increase in steps of `100`
- An `All` option is shown when total records do not exactly match the last 100-step option

Examples:

- For 250 rows, page size options will be `100`, `200`, and `All`
- For 1,000 rows, page size options will be `100` through `1000`

## Opening Full Row Details

In the `Card Details` column, each issue key is clickable.

When clicked:

- A modal window opens
- The modal shows the full row in a card-style layout
- This is useful when description text or links are too long to review comfortably in the table

To close the detail view:

- Click the `Close` button
- Click outside the modal
- Press `Esc`

## Links in the Table

The `Business Docs` and `Tutorial Doc` columns automatically render clickable links when the value looks like a valid URL.

- Clicking a link opens it in a new browser tab
- Non-URL values are shown as plain text

## Reset Filters

The `Reset Filters` button clears the current filter selections and the global search.

After reset:

- All dropdown filters are cleared first
- The `Release Month` default is applied again
- The global search box is cleared

This means reset does **not** always show all records. If the previous month exists in the data, that month is automatically selected again.

## Download Excel

The `Download Excel` button exports the rows currently visible after filters and search are applied.

Export behavior:

- Only filtered/search-matching rows are exported
- `Issue Type` is excluded from the Excel output
- Column names in the export use the display labels shown in the UI
- The file name includes the selected release month values when available

Example file names:

- `release-notes-Mar-2026.xlsx`
- `release-notes-All.xlsx`

If no rows match the current filters, export will not proceed.

## What First-Time Users Should Know

- The page may not initially show all data because `Release Month` defaults to the previous month.
- `Sr. No.` is dynamic and changes with the current view.
- Search and filters work together.
- Reset returns the table to the app default state, not necessarily a fully unfiltered state.
- The detail modal is the best place to inspect a full issue entry.
- Excel export always reflects the current filtered result set.

## Running the App Locally

The app expects to load data from `./data/master.csv`.

### Recommended approach

Run the page through a local web server, for example using:

- `start-local-server.ps1`
- `start-local-server.bat`

### If the file is opened directly

If `index.html` is opened using `file://`, some browsers block `fetch()` access to the CSV.

In that case, the app will show an `Upload CSV File` button so the user can manually select `data/master.csv`.

## Common User Workflow

1. Open the page.
2. Check whether `Release Month` is already pre-filtered.
3. Adjust filters as needed.
4. Use global search for keywords or issue keys.
5. Sort a column if you want a different view.
6. Open `Card Details` to inspect a specific row.
7. Export the current result set to Excel if needed.

## Known Behavior Notes

- Records with blank or invalid release dates are treated as lowest-priority in the default ordering.
- `Release Month` filtering is based on parsed month-year values, not the raw original text.
- Search, filter, and sorting all affect the visible row numbering.
